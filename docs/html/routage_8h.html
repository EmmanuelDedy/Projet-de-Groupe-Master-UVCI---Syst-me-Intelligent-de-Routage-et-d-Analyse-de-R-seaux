Chemin * routage_dijkstra ( Graphe * g,

Algorithme de Dijkstra (Routage Optimal).

Trouve le chemin minimisant le critère donné (cout ou latence). Utilise une stratégie gloutonne par relaxation.

Pseudo-code :

Pour v dans V: dist[v] = INF, pred[v] = NULL
dist[source] = 0
Q = PriorityQueue(V)
Tant que Q non vide:
u = ExtractMin(Q)
Pour chaque voisin v de u:
alt = dist[u] + poids(u,v)
Si alt < dist[v]:
dist[v] = alt
pred[v] = u
Update(Q, v, alt)
Paramètres
gLe graphe
sourceID nœud départ
destinationID nœud arrivée
critere"cout" ou "latence"
Renvoie
Chemin* Structure allouée ou NULL si inaccessible.

Génère K chemins alternatifs pour la redondance (Failover). Utilise Dijkstra de manière itérative en "supprimant" virtuellement des liens.
